RGS Core Demo UI

Start a session and spin a demo slot game against the rgs-core API running on localhost:8000.

1. Session

Create a demo session for an operator + player. The backend initializes the player with a fake balance.

No active session
No session yet.

2. Game Spin

Use the active session to spin a demo game. A simple RNG decides if the round wins or loses.

Balance: –
Spins will fail if you don’t have an active session.

3. API Targets

This UI assumes your backend container is running on http://localhost:8000 with the routes we defined.

GET /health POST /session/start POST /games/{game_id}/spin

You can hit these directly with curl or Postman as well. The UI is just a thin wrapper for interviews and quick demos.
